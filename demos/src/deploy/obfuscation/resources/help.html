This demo shows how to obfuscate an yFiles for JavaFX application. The obfuscation is done using yGuard, the free bytecode obfuscator that is available at https://www.yworks.com/products/yguard.

Obfuscation

Obfuscation changes class, field, and method names in a consistent but nonsensical way that does not change the functionality of the software but makes decompiled Java code harder to read and publicly available API description of the original code inapplicable to the obfuscated code. Obfuscation is an important step in the process of deploying an application in order to protect your software. This demo also shows how to obfuscate the yFiles library, which is required per license agreement for deploying an application that uses it.

Since some names need to be kept at runtime in order for Java(FX) applications to work, yGuard provides several mechanisms to control the obfuscation process. In this demo, annotations are used for this. For a fully documentation of the annotation mechanism click on the yGuard tab above or visit our website at www.yworks.com.

The Demo Application

A simple GraphControl is shown that has a JavaFX rectangle in the background. The rectangle is positioned and sized to match the content rectangle of the GraphControl. This is entirely done using fxml and data binding.

Things to try

  • Create new nodes and see how the background rectangle resizes and repositions accordingly.
  • Pan and zoom in the GraphControl to see that that the background rectangle also zooms and repositions accordingly.
  • Hit the "update content rectangle" button in the toolbar to manually update the GraphControl's content rectangle to the current graph bounds.
  • Run the show ant task to inspect the obfuscation.
  • Switch to the yGuard tab to have a look at the yGuard documentation.
