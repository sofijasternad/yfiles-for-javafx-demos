This demo shows how to obfuscate a yFiles for JavaFX application.
The obfuscation is done using yGuard, the free bytecode obfuscator that is available at github.com/yWorks/yGuard.
Please see the yGuard documentation hosted at yworks.github.io/yGuard for further information on how to configure and use yGuard.

Obfuscation

Obfuscation changes class, field, and method names in a consistent but nonsensical way that does not change the functionality of the software but makes decompiled Java code harder to read and publicly available API description of the original code inapplicable to the obfuscated code. Obfuscation is an important step in the process of deploying an application in order to protect your software. See how to obfuscate the yFiles library.

Since some names need to be kept at runtime in order for Java(FX) applications to work, yGuard provides several mechanisms to control the obfuscation process. In this demo, annotations are used for this. Please see Controlling obfuscation exclusion with annotations in the yGuard documentation for further information.

The Demo Application

A simple GraphControl is shown that has a JavaFX rectangle in the background. The rectangle is positioned and sized to match the content rectangle of the GraphControl. This is entirely done using FXML and data binding.

Things to try

  • Create new nodes and see how the background rectangle resizes and repositions accordingly.
  • Pan and zoom in the GraphControl to see that the background rectangle also zooms and repositions accordingly.
  • Hit the "update content rectangle" button in the toolbar to manually update the GraphControl's content rectangle to the current graph bounds.
  • Run the show ant task to inspect the obfuscation.
