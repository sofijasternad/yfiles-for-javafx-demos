Step 8: Using GraphML for Loading and Saving

Graphs can be saved and loaded into an application using the GraphML format.

GraphML is the standard file format of yFiles. It is an XML format that allows for great flexibility when storing custom data. However, note that these attributes (such as styles or even node locations) are not standardized. So you probably won't be able to exchange all of them between different graph libraries, for example.

Class GraphControl already comes with several convenience methods for reading and writing GraphML files.

In addition, the usual commands Open, Save and Save as are available by default but must be enabled once.

If you want to customize your I/O process, e.g., to write additional custom data, see class GraphMLIOHandler and step Export Custom Data in GraphML later in this tutorial.

Things to Try

  • To save a graph, create one and select Save... in the menu File.
  • To load a graph, select Open... in the menu File.

See the sources for details.
